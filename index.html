Welcome


My name is Nicklas and I am an educated UI/UX Designer and Information Architect (Msc IT). After graduating my master's program in the summer of 2021, I started to dig into JavaScript to strenghten my existing knowledge of HTML/CSS and get a better overall understanding of web development.

Originally this approach was meant as a way to become better at communicating with developers. However, I started to see a natural connection between my existing design/architecture knowledge and what I was capable of building with JavaScript frameworks like React. I decided that this was what I wanted to pursue a career in.

Motivation


I am motivated to build digital products that users find great and intuitive. Something that users keep coming back to again and again. What initially pushed me towards web design was the extreme amount of unfriendly web experiences. I wanted to do something about that.


More recently my motivation has been more development-focused. I think the technology in web development is incredibly interesting to utilize in a practical setting and I could easily see myself as a full-stack developer at one point in my career. However, right now my focus is on the frontend and how to manage it properly.

Skillset


The toolbox of practical theories and methodology that I have acquired throughout my studies has a broad scope in terms of applicability. Subjects such as design psychology, audience analysis, media sociology, test strategies, data handling, wireframing etc. all ties into my overall skillset and validity in terms of participating in design and architecture.


Since the summer of 2021 I have been working on becoming a frontend (React.js) developer. Although I did possess a modest understanding of HTML/CSS beforehand, I still had no real experience with programming. I quickly found Codecademy's Frontend Career Path which helped me get started with a structured and up-to-date curriculum.

After getting to know the basics of JavaScript and React I began to look elsewhere for learning resources. Especially platforms like Udemy and YouTube helped me dive into web technologies such as Redux, Next.js, Material UI, Sass and Firebase + many more. Now I have reached a point where I see myself in a junior frontend position.

Movie Database


This is a simple application made with React. It fetches and displays data about movies and tv shows from a publicly accessible API. Filtering is achieved with a search function. The project showcases fundamental React understanding with hooks such as useState and useEffect. It also shows dictation of rendered content based on ternary operators. The app might bug on fetching if you type too fast.







Goal Catcher


This is a basic CRUD application made with the MERN stack. It is fully functional with an Express API and MongoDB. Authentication has been made secure with JSON Web Tokens so that passwords are not stored directly in the database. Redux has been utilized in the frontend to manage state. For easy access use the email test@test.com with password to log into the app.







Expense Tracker


This is a React budgeting tool made with Material UI and Chart.js in order to deliver pitch perfect user experience. State is managed by utilizing the useContext hook. There is also a custom React hook present for managing transactions. Local storage makes it possible to render the users previous transactions and thus make the app useable in a real setting.







Note Keeper


This is a very basic and minimal vanilla todo application. It's extremely lightweight since it doesn't require a lot of code to deliver the experience that most people would look for in this kind of application. Things can quickly become complicated when making digital products with frameworks and libraries but sometimes its better to keep things simple and predictable.
